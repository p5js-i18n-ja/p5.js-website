  <p>Divides a vector's <code>x</code>, <code>y</code>, and <code>z</code>
  components by the same number, separate numbers, the components of another <a
  href="#/p5.Vector">p5.Vector</a> object, or an array of numbers. Calling
  <code>div()</code> with no arguments has no effect.</p>

  <p>The static version of <code>div()</code>, as in <code>p5.Vector.div(v,
  2)</code>, returns a new <a href="#/p5.Vector">p5.Vector</a> object and
  doesn't change the originals.</p>


  <hr/>

  <p>Divides a vector's <code>x</code>, <code>y</code>, and <code>z</code>
  components.</p>

  <p><code>div()</code> can use separate numbers, as in <code>v.div(1, 2,
  3)</code>, another

  <a href="/reference/p5/p5.Vector/">p5.Vector</a> object, as in
  <code>v.div(v2)</code>, or an array

  of numbers, as in <code>v.div([1, 2, 3])</code>.</p>

  <p>If only one value is provided, as in <code>v.div(2)</code>, then all the
  components

  will be divided by 2. If a value isn't provided for a component, it

  won't change. For example, <code>v.div(4, 5)</code> divides <code>v.x</code>
  by, <code>v.y</code> by 5,

  and <code>v.z</code> by 1. Calling <code>div()</code> with no arguments, as in
  <code>v.div()</code>, has

  no effect.</p>

  <p>The static version of <code>div()</code>, as in <code>p5.Vector.div(v,
  2)</code>, returns a new

  <a href="/reference/p5/p5.Vector/">p5.Vector</a> object and doesn't change the

  originals.</p>
